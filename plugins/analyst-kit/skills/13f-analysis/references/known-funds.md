## ⚠️ / ❌ notes — the traps

- **Greenlight / Einhorn** → file under **DME Capital Management, LP** (`0001489933`),
  *not* "Greenlight Capital Inc" (`0001079114`), whose 13F filings **stopped after
  period 2023-12-31**. Confirmed via holdings (Green Brick, Brighthouse, Fluor).
- **Baupost** → **BAUPOST GROUP LLC/MA** (`0001061768`) is the live filer.
  "BAUPOST GROUP LLC /ADV" (`0001054420`) is a stale shell (last 13F 2002). Note:
  Baupost reports **values in $thousands** — `fetch_13f.py` auto-normalizes.
- **Kuppy** → **Praetorian PR LLC** (`0001949877`) — Kupperman is in Puerto Rico
  ("PR"). Confirmed via signature holdings (St. Joe, Sprott, Valero, Marex). Do
  **not** use "Praetorian Capital Management LLC" (`0001337851`, stopped 2023) or
  "Praetorian Wealth Management, Inc." (`0001818940`, an unrelated ETF-holding RIA).
- **Duquesne** → **Duquesne Family Office LLC** (`0001536411`) is live;
  "Duquesne Capital Management" (`0001008925`) is Druckenmiller's wound-down hedge
  fund (last 13F 2010).
- **Citrini Research** is a research/Substack outfit — **not a registered manager
  and files no 13F.** There is nothing to pull from EDGAR.
- **Duquesne** also reports **values in $thousands** (auto-normalized).

## Refresh all of them in one go

```bash
cd skills/13f-analysis/scripts
export SEC_EDGAR_UA="your-app your-name you@example.com"
for cik in 0001067983 0001336528 0001536411 0001656456 0001040273 \
           0001345471 0001056831 0001489933 0000921669 0001061768 \
           0001647251 0001103804 0001167483 0001709323 0001135730 \
           0001061165 0001949877; do
  python3 fetch_13f.py "$cik" --quiet --out ../../../13f-output
done
```

> CIKs are stable identifiers, but a manager can change filing entity over time
> (as Greenlight did). If a "latest" filing looks stale, re-run `find_fund.py` on
> the firm name to discover the current entity.
