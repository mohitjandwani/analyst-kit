# <Sector> playbook

Copy this file to `<sector>.md` and fill every section. Keep it dense — each line
should prevent a concrete mistake or point at the number that actually matters.

## Reporting calendar & conventions

How companies in this sector define fiscal years/quarters; label conventions
(does "FY2025" mean the year ending in 2025 or starting in it?); any non-quarterly
disclosure cadence (monthly sales, weekly bookings).

## KPIs that matter

The 5-8 metrics professionals actually track, with where each is disclosed
(fundamentals API vs earnings release vs filing) and how it's defined.

## Normalization rules

What must be adjusted before any comparison: calendar re-binning, currency,
per-unit measures, definition changes over time.

## Seasonality

Which quarter dominates and why; what sequential comparisons are meaningless.

## Valuation norms

The multiples/approaches the market prices this sector on, and the regime in which
each applies (growth vs cyclical trough, etc.).

## Common traps

Mistakes that produce confidently wrong analysis in this sector.

## Data-source quirks

Which of this repo's skills carries which series for this sector, and known gaps
(e.g. KPIs that exist only in 8-K exhibits).
